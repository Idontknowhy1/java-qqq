# jikeing-web 接口整理

本文档整理了当前项目 src/api 下的后端接口封装，并补充了对应的调用页面或功能场景，便于统一查看和与后端对齐。

## 一、请求基础说明

- 请求封装位于 src/api/request.js。
- 默认请求基地址为 https://api.jikeing.com/。
- 当运行环境域名变化时，会按当前 host 自动切换 api 子域名。
- 请求会自动携带本地存储中的 userToken，并写入 x-token 请求头。
- get 请求会把 data 转成 query params；post 和 put 直接发送 body。
- 项目中还使用了两个环境变量前缀：VITE_JAVA_BASE_API 和 VITE_JAVA_ADMIN_BASE_API。

## 四、补充说明

- src/api/roleList.js 和 src/api/actionList.js 提供的是本地静态素材数据，不是后端接口，但会被 AI 生成相关页面当作角色和动作素材库使用。
- 当前清单以项目源码中可检索到的封装为准；如果后端还有未封装或通过字符串拼接调用的接口，需要继续在页面代码里单独排查。
